Useful and simple IPtables rules
1. Basic IPtables rules for Linux

Check the default policy chain behavior

$ sudo iptables -L | grep policy

Check the current rules

$ sudo iptables -L

List rules by specification

$ sudo iptables -S

Check your Iptables status

$ sudo iptables -L -v

Saving modified Iptables

$ sudo service iptables save

Flush Iptables and persist changes

$ sudo iptables -F && sudo /sbin/iptables-save

2. Administration of Linux IPtables

Check existing rules for specific tables

$ sudo iptables -t nat -L -v -n

List rules for TCP chains only

$ sudo iptables -S TCP

3. Linux IPtables Firewall Rules

Block all incoming requests

$ sudo iptables INPUT -j DROP

Block a specific IP address

$ sudo iptables -A INPUT -s xxx.xxx.xxx.xxx -j DROP

Block all TCP requests from an IP

$ sudo iptables -A INPUT -p tcp -s xxx.xxx.xxx.xxx -j DROP

Unblock an IP address

$ sudo iptables -D INPUT -s xxx.xxx.xxx.xxx -j DROP

Block IP address ranges

$ sudo iptables -A INPUT -s xxx.xxx.xxx.0/24 -j DROP

Unblock IP address ranges

$ sudo iptables -D INPUT -s xxx.xxx.xxx.0/24 -j DROP

Block all TCP requests for given IP range

$ sudo iptables -A INPUT -p tcp -s xxx.xxx.xxx.0/24 -j DROP

Unblock all TCP requests for Given IP Range

$ sudo iptables -D INPUT -p tcp -s xxx.xxx.xxx.0/24 -j DROP

Block TCP connections on specific ports

$ sudo iptables -A OUTPUT -p tcp --dport 111 -j DROP

$ sudo iptables -A INPUT -p tcp --dport xxx -j DROP

Allow TCP connections on port 80

$ sudo iptables -A INPUT -p tcp -s xxx.xxx.xxx.0/24 --dport 80 -j ACCEPT

Reject TCP connections on port 80

$ sudo iptables -A INPUT -p tcp -s xxx.xxx.xxx.0/24 --dport 80 -j DROP

$ sudo iptables -A INPUT -p udp -s xxx.xxx.xxx.0/24 --dport 80 -j DROP

Allow incoming SSH connections on port 22

$ sudo iptables -A INPUT -p tcp -s xxx.xxx.xxx.0/24 --dport ssh -j ACCEPT

Establish states when allowing incoming SSH

$ sudo iptables -A INPUT -i eth0 -p tcp --dport 22 -m state --state NEW,ESTABLISHED -j ACCEPT

Allow multiple ports for incoming requests

$ sudo iptables -A INPUT -p tcp -m multiport --dports 22,80,110 -j ACCEPT

Block Facebook in Iptables rules

Often many companies block social media sites such as Facebook during office hours. The below commands can be used for this purpose. First, find out the network range used by Facebook in your geo-location.

$ sudo host facebook.com

This should return a result consisting of the specific IP used by Facebook, say 157.240.7.35 in this case. Now run the next command.

$ sudo whois 66.220.156.68 | grep CIDR

It will provide the IP range Facebook uses for your location, say 157.240.0.0/16 in this case. Now we can simply block all outgoing connections to this network.

$ sudo iptables -A OUTPUT -p tcp -d 157.240.0.0/16 -j DROP

Block incoming ping requests

$ sudo iptables -A INPUT -pr icmp -i eth0 -j DROP

Log dropped network packets

$ sudo iptables -A INPUT -i eth0 -j LOG --log-prefix "IPtables dropped packets:"

4. Miscellaneous IPtables firewall rules

Allow port forwarding in Iptables

$ sudo iptables -t nat -A PREROUTING -i eth0 -p tcp --dport 25 -j REDIRECT --to-port 3535

Allow loopback access

$ sudo iptables -A INPUT -i lo -j ACCEPT $ sudo iptables -A OUTPUT -o lo -j ACCEPT

Block access to specific MAC addresses

$ sudo iptables -A INPUT -m mac --mac-source 00:00:00:00:00:00 -j DROP

Limit concurrent connections for per IP

$ sudo iptables -A INPUT -p tcp --syn --dport 22 -m connlimit --connlimit-above 3 -j REJECT

Save Iptables rules to a file

$ sudo iptables-save > ~/iptables.rules

Restore Iptables from a file

$ sudo iptables-restore < ~/iptables.rules
