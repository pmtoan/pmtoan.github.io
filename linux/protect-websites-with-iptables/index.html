Protect websites with Iptables

Backup current Iptables rules

$ sudo iptables-save > Backup_IPTables`date "+%Y-%m-%d_%H:%M:%S"`.old

Clear all rules and chains

$ sudo iptables -F
$ sudo iptables -X

Allow loopback connections

$ sudo iptables -A INPUT -i lo -j ACCEPT
$ sudo iptables -A OUTPUT -o lo -j ACCEPT

Allow connection already established

$ sudo iptables -A INPUT -i enp0s3 -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
$ sudo iptables -A OUTPUT -o enp0s3 -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

Allow ping

$ sudo iptables -A INPUT -i enp0s3 -p icmp -m icmp --icmp-type any -j ACCEPT
$ sudo iptables -A OUTPUT -o enp0s3 -p icmp -m icmp --icmp-type any -j ACCEPT

Allow DNS queries and accesses

$ sudo iptables -A OUTPUT -o enp0s3 -p udp --dport 53 --sport 1024:65535 -m conntrack --ctstate NEW -j ACCEPT
$ sudo iptables -A OUTPUT -o enp0s3 -p tcp --dport 53 --sport 1024:65535 -m conntrack --ctstate NEW -j ACCEPT

Allow NTP date update

$ sudo iptables -A OUTPUT -o enp0s3 -p udp --sport 123 --dport 123 -m conntrack --ctstate NEW -j ACCEPT

Allow only connection to specify port and specify source address. Example log and allow only SSH port 22 from IP 192.168.56.1.

$ sudo iptables -A INPUT -i enp0s3 -p tcp --dport 22 -s '192.168.56.1' -m conntrack --ctstate NEW -j LOG --log-prefix 'SSH connection from 192.168.56.1: '
$ sudo iptables -A INPUT -i enp0s3 -p tcp --dport 22 -s '192.168.56.1' -m conntrack --ctstate NEW,ESTABLISHED -j ACCEPT

Flooding of RST packets, smurf attack rejection.

$ sudo iptables -A INPUT -p tcp -m tcp --tcp-flags RST RST -m limit --limit 2/second --limit-burst 2 -j ACCEPT

Protecting server, block IPs try to connect to port 22 and 63378

# Attacking IP will be locked for 24 hours (3600 x 24 = 86400 Seconds)
$ sudo iptables -A INPUT -m recent --name portscan --rcheck --seconds 86400 -j LOG --log-prefix "IP blacklist try to SSH: "
$ sudo iptables -A INPUT -m recent --name portscan --rcheck --seconds 86400 -j DROP
# Remove attacking IP after 24 hours
$ sudo iptables -A INPUT -m recent --name portscan --remove
# These rules add scanners to the portscan list, and log the attempt.
# Protect for Port SSH 63379
$ sudo iptables -A INPUT -p tcp -m tcp --dport 22 -m recent --name portscan --set -j LOG --log-prefix "SSH 22:"
$ sudo iptables -A INPUT -p tcp -m tcp --dport 22 -m recent --name portscan --set -j DROP
$ sudo iptables -A INPUT -p tcp -m tcp --dport 63378 -m recent --name portscan --set -j LOG --log-prefix "SSH 63378:"
$ sudo iptables -A INPUT -p tcp -m tcp --dport 63378 -m recent --name portscan --set -j DROP

Reject SQL injection

$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27+or+%271%27%3d%271" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27+or+1%3d1" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27+or+%271%27%3d%271" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27)+or+%27%27%3d%27" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27)+or+1%3d1" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27)+or+%271%27%3d%271" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27)+or+(%27%27%3d%27" --algo bm -j LOG
$ sudo iptables -I INPUT -s 0.0.0.0/0 -m string --string "%27)+or+(1%3d1" --algo bm -j LOG
